Linux Tcp 状态 以及 sysctl.conf 配置参数

说明： 通常情况下:一个正常的TCP连接，都会有三个阶段:1、TCP三次握手;2、数据传送;3、TCP四次挥手

里面的几个概念：

  • SYN: (同步序列编号,Synchronize Sequence Numbers)
  • ACK: (确认编号,Acknowledgement Number)
  • FIN: (结束标志,FINish)


####TCP三次握手(创建 OPEN)

  1. 客户端发起一个和服务创建TCP链接的请求，这里是SYN(J)
  2. 服务端接受到客户端的创建请求后，返回两个信息： SYN(K) + ACK(J+1)
  3. 客户端在接受到服务端的ACK信息校验成功后(J与J+1)，返回一个信息：ACK(K+1)
  4. 服务端这时接受到客户端的ACK信息校验成功后(K与K+1)，不再返回信息，后面进入数据通讯阶段

数据通讯

  1. 客户端/服务端 read/write数据包

TCP四次握手(关闭 finish)

  1. 客户端发起关闭请求，发送一个信息：FIN(M)
  2. 服务端接受到信息后，首先返回ACK(M+1),表明自己已经收到消息。
  3. 服务端在准备好关闭之前，最后发送给客户端一个 FIN(N)消息，询问客户端是否准备好关闭了
  4. 客户端接受到服务端发送的消息后，返回一个确认信息: ACK(N+1)
  5. 最后，服务端和客户端在双方都得到确认时，各自关闭或者回收对应的TCP链接。

####详细的状态说明(以及linux相关参数调整)

/etc/sysctl.conf

  1. SYN_SEND
    1. 客户端尝试连接服务端，通过open方法。也就是TCP三次握手中的第一步后, 客户端状态
    2. sysctl -w net.ipv4.tcp_syn_retries = 2 作为客户端可以设置SYN包的重试次数，默认5次(大约180s)
  2. SYN_RECEIVED
    1. 服务接收创建请求的SYN之后，也就是TCP三次握手的第二步，发送ACK数据包之前。
    2. 注意服务端状态，一般15个左右正常，如果很大，怀疑遭受SYN_GLOOD攻击。
    3. sysctl -w net.ipv4.tcp_max_syn_backing = 4096, 设置该状态的等待队列数，默认1024，调大后可以适当放置syn_flood, 可参见 man 7 tcp.
    4. sysctl -w net.ipv4.tcp_syncookies = 1 ，打开syn backing队列不足的时候，提供一种机制临时将syn链接换出
    5. sysctl -w net.ipv4.tcp_syn_retries = 2 作为客户端可以设置SYN包的重试次数，默认5次(大约180s)
  3. ESTABLISHED
    1. 客户端接受到服务端的ACK包后的状态，服务端在发出ACK在一定时间后即为 ESTABLISHED
    2. sysctl -w net.ipv4.tcp_keepalive_time = 1200 默认是7200秒(2小时). 系统针对空闲链接会进行心跳检查，如果超过 net.ipv4.tcp_keepalive_probes * net.ipv4.tcp_keepalive_intvl = 默认 11 分,终止对应的tcp链接，可适当调整心跳检查频率。
    3. 目前线上的监控waring:600 , critial : 800
  4. FIN_WAIT1
    1. 主动关闭的一方，在发出FIN请求之后，也就是TCP四次挥手的第一步
  5. CLOSE_WAIT
    1. 被关闭的一方，在接受到客户端的FIN后，也就是在TCP四次挥手的第二步。
  6. FIN_WAIT2
    1. 主动关闭的一方，在接受到被动关闭一方的ACK后，也就是TCP四次挥手的第二步。
    2. sysctl -w net.ipv4.tcp_fin_timeout=30, 可以设定被动关闭方返回FIN后的超时时间，有效回收链接，避免syn-flood.
  7. LASK_ACK
    1. 被关闭一方，在发送ACK后一段时间后(确保客户端已经收到), 在发起一个FIN请求。也就是TCP四次握手第三步
  8. TIME_WAIT
    1. 主动关闭的一方，在收到被动关闭的FIN包后，发送ACK。这就是TCP四次握手的第四部。
    2. sysctl -w net.ipv4.tcp_tw_recycle = 1 打开快速回收TIME_WAIT。Enabling this option is not recommended since this causes problems when working with NAT (Network Address Translation)
    3. sysctl -w net.ipv4.tcp_tw_reuse = 1 快速回收并重用TIME_WAIT的链接。
    4. net.ipv4.tcp_max_tw_buckets 处于time_wait状态的最多连接数， 默认为 180000。

补充：

  1. 主动关闭方在接收到被动关闭放的FIN请求后，发送成功给对方一个ACK后，将自己的状态由FIN_WAIT2 修改为TIME_WAIT， 而必须再等2倍的MSL(Maximun Segment Lifetime, MSL是一个数据报在internetwork中能存在的时间) 时间之后，双方才能把状态都改为CLOSED以关闭链接。目前RHEL中保持TIME_WAIT状态的时间为60s。
  2. keepalive策略可以有效避免进行三次握手和四次挥手的动作。
